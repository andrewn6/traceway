title: Datasets and Evaluations
description: Build test datasets from production data and run evaluations to measure model quality.

## The workflow

1. Your app sends traces to Traceway.
2. You spot an interesting span in the dashboard (or set up a capture rule to save them automatically).
3. Export the span's input/output to a dataset.
4. Once you have enough datapoints, run an evaluation against a model.
5. Compare eval runs to see how changes to your prompt or model affect quality.

This turns production data into regression tests.

## Creating a dataset

```ts
const tw = new Traceway({ url: '...', apiKey: '...' });

const dataset = await tw.createDataset('qa-golden-set', 'Curated Q&A pairs');
```

Or via the API:

```bash
curl -X POST https://api.traceway.ai/api/datasets \
  -H "Authorization: Bearer tw_sk_..." \
  -H "Content-Type: application/json" \
  -d '{"name": "qa-golden-set", "description": "Curated Q&A pairs"}'
```

## Adding datapoints

### Manually

```ts
await tw.createDatapoint(dataset.id, {
  Generic: {
    input: { question: 'What is the capital of France?' },
    expected_output: { answer: 'Paris' },
  },
});
```

### From a production span

In the dashboard, click a span and hit "Export to Dataset." Or programmatically:

```ts
await tw.exportSpanToDataset(dataset.id, spanId);
```

This copies the span's input and output into a new datapoint.

### From a file

Upload a JSON, JSONL, or CSV file:

```bash
curl -X POST "https://api.traceway.ai/api/datasets/${DATASET_ID}/import" \
  -H "Authorization: Bearer tw_sk_..." \
  -F "file=@testcases.jsonl"
```

JSONL format (one object per line):

```json
{"input": {"question": "What is 2+2?"}, "expected_output": {"answer": "4"}}
{"input": {"question": "Capital of Japan?"}, "expected_output": {"answer": "Tokyo"}}
```

### With capture rules

Capture rules save spans to a dataset automatically when they match a filter.

Set up a rule in the dashboard under your dataset's "Capture Rules" tab:
- **Name**: `expensive-gpt4-calls`
- **Filter**: model = `gpt-4o`, cost > $0.05
- **Sample rate**: 0.1 (save 10% of matching spans)

Every time a span completes that matches the filter, Traceway creates a datapoint in the target dataset (if the random sample passes).

## Running an evaluation

An eval takes every datapoint in a dataset, sends the input to a model, and compares the output against the expected value.

### Prerequisites

You need a **provider connection** — credentials for the model you want to evaluate. Set this up in the dashboard under Settings > Provider Connections, or via the API:

```bash
curl -X POST https://api.traceway.ai/api/provider-connections \
  -H "Authorization: Bearer tw_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "openai-prod",
    "provider": "openai",
    "api_key": "sk-...",
    "default_model": "gpt-4o-mini"
  }'
```

### Starting a run

```bash
curl -X POST "https://api.traceway.ai/api/datasets/${DATASET_ID}/eval" \
  -H "Authorization: Bearer tw_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "gpt-4o-mini baseline",
    "config": {
      "connection_id": "CONNECTION_ID",
      "model": "gpt-4o-mini",
      "system_prompt": "Answer concisely.",
      "temperature": 0.0
    },
    "scoring": "ExactMatch"
  }'
```

Scoring strategies:

| Strategy | How it scores |
|----------|--------------|
| `ExactMatch` | Output must equal `expected_output` exactly. Score is 1.0 or 0.0. |
| `Contains` | Output must contain `expected_output` as a substring. Score is 1.0 or 0.0. |
| `LlmJudge` | A second LLM call evaluates whether the output is correct. Score is 0.0-1.0. |
| `None` | No scoring. Just records the outputs for manual review. |

### Checking progress

Evals run asynchronously. Poll the run:

```bash
curl "https://api.traceway.ai/api/eval/${RUN_ID}" \
  -H "Authorization: Bearer tw_sk_..."
```

The `status` field will be `running`, `completed`, `failed`, or `cancelled`. The `progress` and `total` fields tell you how far along it is.

### Comparing runs

Run the same dataset with different models or prompts, then compare:

```bash
curl "https://api.traceway.ai/api/datasets/${DATASET_ID}/compare?runs=${RUN_1},${RUN_2}" \
  -H "Authorization: Bearer tw_sk_..."
```

Returns a table of every datapoint with each run's output and score side by side.

## Human review queue

For datapoints that need human judgment, use the review queue:

1. Enqueue datapoints: `POST /api/datasets/:id/queue` with a list of datapoint IDs
2. Reviewers claim items: `POST /api/queue/:item_id/claim`
3. Reviewers submit corrections: `POST /api/queue/:item_id/submit`

This is useful for building high-quality training data, reviewing edge cases from capture rules, or validating model outputs before they go into your golden set.
