## Local development

The fastest way to get started. No Docker, no cloud account.

```bash
cargo install traceway
traceway serve
```

This starts:
- API server on `http://localhost:3000`
- Proxy on `http://localhost:3001` (if you want auto-instrumentation)

Storage is SQLite (written to `./traceway.db`). Auth is disabled. Nothing leaves your machine.

## Docker

```bash
docker run -p 3000:3000 -p 3001:3001 ghcr.io/blastgits/traceway:latest
```

Or with docker-compose:

```yaml
version: '3'
services:
  traceway:
    image: ghcr.io/blastgits/traceway:latest
    ports:
      - "3000:3000"
      - "3001:3001"
    volumes:
      - traceway-data:/data
    environment:
      - TRACEWAY_DATA_DIR=/data

volumes:
  traceway-data:
```

## Cloud mode

For production, run with `--cloud` to enable auth and use Postgres + Turbopuffer:

```bash
traceway serve --cloud
```

Required environment variables:

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | Postgres connection string for auth data |
| `TURBOPUFFER_API_KEY` | Turbopuffer API key for trace/span storage |
| `JWT_SECRET` | Secret for signing session tokens (min 32 chars) |
| `RESEND_API_KEY` | Resend API key for transactional emails (invites, password resets) |
| `POLAR_ACCESS_TOKEN` | Polar.sh token for billing (optional) |
| `POLAR_WEBHOOK_SECRET` | Polar webhook HMAC secret (optional) |

### Deploy to Railway

Traceway is deployed on Railway at `api.traceway.ai`. To run your own instance:

1. Create a new project on [Railway](https://railway.app)
2. Add a Postgres database
3. Deploy the Docker image `ghcr.io/blastgits/traceway:latest`
4. Set the environment variables above
5. Expose port 3000

The API binary handles its own HTTPS termination via Railway's proxy.

### Environment details

- **Ports**: API on 3000, Proxy on 3001
- **Health check**: `GET /api/health` returns `{"status": "ok", ...}`
- **Readiness probe**: `GET /api/ready`
- **Liveness probe**: `GET /api/live`

## Storage architecture

| Mode | Auth | Traces/Spans | Datasets/Evals |
|------|------|-------------|----------------|
| Local (`traceway serve`) | Disabled | SQLite | SQLite |
| Cloud (`traceway serve --cloud`) | Postgres | Turbopuffer | Turbopuffer |

In cloud mode, each project gets its own Turbopuffer namespace (`tw_{org}_{project}`). Auth data (users, orgs, API keys, invites) is always in Postgres.

## Plans and limits

| Plan | Spans/month | Members | Retention |
|------|-------------|---------|-----------|
| Free | 10,000 | 1 | 7 days |
| Pro ($20/user/mo) | 1,000,000 | 5 | 30 days |
| Team ($100/user/mo) | 10,000,000 | 50 | 90 days |
| Enterprise | Unlimited | Unlimited | Custom |

Span limits are enforced per-organization. When you hit the limit, new spans are rejected with `402 Payment Required`.

## Building from source

```bash
git clone https://github.com/blastgits/traceway.git
cd traceway
cargo build --release -p api
./target/release/traceway serve
```

Note: Building the full workspace requires macFUSE for the `fuser` crate. To skip that:

```bash
cargo build --release -p api -p proxy -p trace
```
