## Datasets

### Create a dataset

```
POST /api/datasets
```

```json
{
  "name": "golden-set",
  "description": "Curated test cases for the Q&A pipeline"
}
```

### List datasets

```
GET /api/datasets
```

Returns datasets with their datapoint counts:

```json
{
  "datasets": [
    {
      "id": "01J...",
      "name": "golden-set",
      "description": "Curated test cases",
      "datapoint_count": 42,
      "created_at": "2024-06-15T12:00:00Z",
      "updated_at": "2024-06-15T14:30:00Z"
    }
  ],
  "count": 1
}
```

### Update a dataset

```
PUT /api/datasets/:id
```

```json
{
  "name": "golden-set-v2",
  "description": "Updated test cases"
}
```

### Delete a dataset

```
DELETE /api/datasets/:id
```

## Datapoints

### Create a datapoint

```
POST /api/datasets/:id/datapoints
```

```json
{
  "kind": {
    "Generic": {
      "input": { "question": "What is the capital of France?" },
      "expected_output": { "answer": "Paris" }
    }
  }
}
```

Or for conversation-style datapoints:

```json
{
  "kind": {
    "LlmConversation": {
      "messages": [
        { "role": "user", "content": "What is 2+2?" }
      ],
      "expected": "4"
    }
  }
}
```

### Export a span to a dataset

```
POST /api/datasets/:id/export-span
```

```json
{
  "span_id": "01J..."
}
```

Creates a datapoint from the span's input/output.

### Import from file

```
POST /api/datasets/:id/import
Content-Type: multipart/form-data
```

Upload a CSV, JSON, or JSONL file. Each row/object becomes a datapoint.

**JSON format**: An array of objects with `input`, `expected_output`, and optionally `messages`.

**JSONL format**: One JSON object per line.

**CSV format**: Columns are mapped to datapoint fields.

Returns:

```json
{
  "imported": 150,
  "dataset_id": "01J..."
}
```

### List datapoints

```
GET /api/datasets/:id/datapoints
```

### Delete a datapoint

```
DELETE /api/datasets/:id/datapoints/:dp_id
```

## Queue

### Enqueue datapoints

```
POST /api/datasets/:id/queue
```

```json
{
  "datapoint_ids": ["01J...", "01J..."]
}
```

### Claim a queue item

```
POST /api/queue/:item_id/claim
```

```json
{
  "claimed_by": "reviewer@example.com"
}
```

Returns `409` if already claimed.

### Submit a reviewed item

```
POST /api/queue/:item_id/submit
```

```json
{
  "edited_data": {
    "corrected_output": "The actual answer is..."
  }
}
```

## Evaluations

### Start an eval run

```
POST /api/datasets/:id/eval
```

```json
{
  "name": "gpt-4o-mini baseline",
  "config": {
    "connection_id": "01J...",
    "model": "gpt-4o-mini",
    "system_prompt": "You are a helpful assistant.",
    "temperature": 0.0
  },
  "scoring": "ExactMatch"
}
```

Scoring options: `ExactMatch`, `Contains`, `LlmJudge`, `None`.

The eval runs asynchronously. Poll `GET /api/eval/:run_id` to check progress.

### Get eval results

```
GET /api/eval/:run_id
```

Returns the run metadata plus all result items:

```json
{
  "id": "01J...",
  "dataset_id": "01J...",
  "name": "gpt-4o-mini baseline",
  "status": "completed",
  "progress": 42,
  "total": 42,
  "result_items": [
    {
      "datapoint_id": "01J...",
      "output": "Paris",
      "score": 1.0,
      "latency_ms": 340,
      "status": "completed"
    }
  ]
}
```

### Compare eval runs

```
GET /api/datasets/:id/compare?runs=run_id_1,run_id_2
```

Returns a side-by-side comparison of outputs and scores for each datapoint across multiple runs.

### Cancel an eval run

```
POST /api/eval/:run_id/cancel
```

### Delete an eval run

```
DELETE /api/eval/:run_id
```
